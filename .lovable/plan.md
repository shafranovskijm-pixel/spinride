
# План: Создание страниц Политика конфиденциальности и Условия использования

## Обзор

Создадим две новые страницы с типовым юридическим текстом для интернет-магазина, следуя существующей архитектуре проекта.

## Файлы для создания

### 1. PrivacyPage.tsx (Политика конфиденциальности)

Новый файл `src/pages/PrivacyPage.tsx` со следующим содержимым:

- Заголовок с иконкой Shield
- Разделы:
  - Общие положения
  - Сбор персональных данных (имя, телефон, email, адрес)
  - Цели обработки данных
  - Защита данных
  - Права пользователя
  - Cookies и аналитика
  - Контакты для обращений

### 2. TermsPage.tsx (Условия использования)

Новый файл `src/pages/TermsPage.tsx` со следующим содержимым:

- Заголовок с иконкой FileText
- Разделы:
  - Общие положения
  - Оформление заказа
  - Оплата и цены
  - Доставка
  - Возврат и обмен
  - Гарантия
  - Ограничение ответственности
  - Изменение условий

## Изменения в существующих файлах

### App.tsx

Добавить:
- Импорт новых страниц
- Два новых маршрута `/privacy` и `/terms`

## Данные в базе

Добавить записи в таблицу `page_content`:

| page_key | title | subtitle | content |
|----------|-------|----------|---------|
| privacy | Политика конфиденциальности | Защита ваших персональных данных | {sections: [...]} |
| terms | Условия использования | Правила использования сайта | {sections: [...]} |

## Структура страниц

Обе страницы будут использовать:
- `ShopLayout` для единообразия с остальным сайтом
- `Card` компоненты для секций
- Загрузка контента через `usePageContent` hook
- Fallback данные на случай отсутствия записи в БД
- Skeleton для состояния загрузки

## Типовой контент

### Политика конфиденциальности
- Соответствие ФЗ-152 "О персональных данных"
- Перечень собираемых данных
- Цели: обработка заказов, доставка, связь
- Права: доступ, изменение, удаление данных
- Сроки хранения данных

### Условия использования
- Правила оформления заказа
- Подтверждение и отмена заказа
- Условия возврата (14 дней, товарный вид)
- Ограничение ответственности магазина
- Порядок разрешения споров

## Результат

После реализации:
1. Страница `/privacy` будет доступна
2. Страница `/terms` будет доступна
3. Ссылки в футере будут работать корректно
4. Администратор сможет редактировать контент через `/admin/pages`
